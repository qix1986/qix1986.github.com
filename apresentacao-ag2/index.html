Animações CSS

Guia nada definitivo sobre o que eu pouco sei delas!

Origens

  • Surgiu graças a crescente demanda por animações em sites, antes desenvolvidas com JavaScript e, antes ainda, com Flash.

Vantagens

  • Não é dependente de script (menos peso na página).
  • É simples de se utilizar do modo mais simples.
  • Tem opções e funções que com JavaScript teria que se ter muito código para replicar.

Transitions

  • É o método mais simples de se realizar uma animação em CSS.
  • Elementos que tenham essa propriedade, ao terem uma ou mais propriedades animará essa diferença utilizando a curva de aceleração, duração e delay estipulados.

exemplo1.html

						
<style>
	#elemento {
		width: 10em;
		height: 10em;
		position: relative;
		background: red;
		opacity: 1;
		transition: all .5s ease-out;
		transform: translate(0);
	}

	#elemento.position {
		transform: translate(200px, 0);
	}

	#elemento.color {
		background: green;
	}

	#elemento.opacity {
		opacity: .3;
	}
</style>
						
					
						
<div id="elemento"></div>
<div class="wrapper-butons">
	<button id="botao-position">Posicao</button>
	<button id="botao-color">Cor</button>
	<button id="botao-opacity">Opacidade</button>
</div>

Animations

  • Fornece um controle maior sobre a animação e o status das propriedades em cada momento da animação.

exemplo2.html

						
<style>
	#elemento {
		width: 10em;
		height: 10em;
		position: relative;
		background: red;
		opacity: 1;
		transform: translate(0);
		-webkit-animation-fill-mode: forwards;
		-webkit-animation-duration: 2s;
		-webkit-animation-timing-function: ease-out;
	}

	#elemento.animate {
		-webkit-animation-name: animacao;
	}
</style>
						
					
						
<style>
	@-webkit-keyframes animacao { 
		33% {
			background: red;
			opacity: 1;
			transform: translate(200px, 0);
		}
		66% {
			opacity: 1;
			background: green;
			transform: translate(400px, 0);
		}
		100% {
			background: green;
			opacity: .3;
			transform: translate(600px, 0);
		}
	}
</style>

Steps

  • Permite que se controle exatamente quantos quadros a animação terá dentro da duração especificada.
  • Indicado para animações fluídas usando sprites.

exemplo3.html

					
<style>
	#frango {
		width: 414px;
		height: 463px;
		display: block;
		background: url(img/lequetreque.png) no-repeat left center;
		-webkit-animation: breathe 1.2s steps(11) infinite alternate;
			animation: breathe 1.2s steps(11) infinite alternate;
	}

	@-webkit-keyframes breathe {
		to { background-position: -4554px; }
	}

	@keyframes breathe {
		to { background-position: -4554px; }
	}
</style>

<div id="frango"></div>

Performance

  • Toda animação mexe deve ser pensada e calculada os custos.
  • Sempre que possível anime propriedades que não causem reflow na página (translate ao invés de position, scale ao invés de width e height...).
  • Procure sempre forçar o browser a utilizar a engine 3D para renderizar as propriedades, com isso você ganha o auxílio da GPU na renderização da página.
  • Veja nos arquivos exemplo4.html (position, width) e exemplo5.html (translate, scale) a diferença de performance nas animações.

Eventos

  • Três tipos de eventos são disparados pelas animações em CSS.
  • animationstart, animationiteration, animationend
  • Transitions possuem esses eventos também, com exceção do iteration.

exemplo6.html

					
<script>
	(function(doc, win, undefined){
		var elemento = doc.getElementById('elemento'),
			botao    = doc.getElementById('botao');

		botao.addEventListener('click', function(){
			elemento.classList.toggle('animate');
		});

		elemento.addEventListener('webkitAnimationStart', function(){
			console.log("Animação começou!")
		});

		elemento.addEventListener('webkitAnimationIteration', function(e){
			console.log("Animação iterou depois de " + e.elapsedTime + " segundos!")
		});

		elemento.addEventListener('webkitAnimationEnd', function(e){
			console.log("Animação terminou depois de " + e.elapsedTime + " segundos!")
		});
	})(document, window);
</script>
